# 중전기 월별 수주/매출 차이분석 프로젝트 보고서

작성일: 2026-06-02

---

## 1. 프로젝트 개요

### 목적

중전기 사업부의 **2026년 1분기(1~3월) 수주 및 매출 실적**이 사업계획 대비 얼마나 차이가 나는지,
그리고 그 원인이 무엇인지를 프로젝트 단위로 추적·분석하는 자동화 파이프라인 구축.

기존에는 Excel에서 수동으로 피벗테이블을 비교하거나 육안으로 항목을 대조해야 했기 때문에
시간이 많이 걸리고 누락이 발생하는 문제가 있었다.
이를 Python 스크립트로 자동화하여 반복 작업을 제거하고 정확도를 높이는 것이 핵심 목표였다.

## 3. 구현 방식

### 기술 스택

| 라이브러리 | 용도 |
|-----------|------|
| pandas / numpy | 데이터 로드, 집계, 비교 |
| chardet | 파일 인코딩 자동 감지 (EUC-KR / UTF-8-sig 혼재) |
| rapidfuzz | 프로젝트명 퍼지 매칭 (유사도 80% 이상) |
| matplotlib | 차트 시각화 (맑은 고딕 한글 폰트) |
| openpyxl | 서식 있는 Excel 보고서 생성 |

### 스크립트 구성

```
분석.py                  ← 핵심: 데이터 로드 → 요약비교 → 프로젝트매칭 → 차트+xlsx 생성
차이원인분석.py           ← 소멸/신규/변동을 구조화한 원인분석 xlsx 생성
서식생성.py              ← 보고서 양식 형태(계획 vs 실적 vs 전년) xlsx 생성
보고기준_서식및매칭.py    ← 보고서 기준 수치 vs CSV 실적 매칭 파일 생성
수치흐름.py / 차이원인.py / 양식생성.py  ← 중간 검증용 보조 스크립트
```

### 처리 파이프라인 (분석.py 기준)

```
Step 1. 데이터 로드 및 전처리
  ├─ 인코딩 자동 감지 (chardet)
  ├─ 사업계획_수주_List: skiprows=9  (상단 9행 = 환율/구성비 헤더)
  ├─ 실적_매출예상:      skiprows=4  (상단 4행 = 환율 헤더)
  ├─ 팀명 정규화:  "01.국내1" → "국내1"
  └─ 제품군 분류:  TR/RT/변압기 → 변압기 / HH/HG/GIS/차단기 → 차단기

Step 2. 요약 비교
  └─ groupby(['월', '팀명', '제품군'])
     → 계획 vs 실적 outer merge
     → 차이 / 달성률 산출

Step 3. 프로젝트 심층 비교
  └─ 매칭 우선순위:
       1순위: 프로젝트명 완전일치
       2순위: PJT번호 일치
       3순위: 발주처 일치
       4순위: 원청 일치
       5순위: 퍼지 매칭 (token_sort_ratio ≥ 80%)
     → 매칭된 건:         금액 변동 분석
     → 계획에만 있는 건:  소멸 (취소/이월 추정)
     → 실적에만 있는 건:  신규 (계획외 수주)

Step 4. 결과물 생성
  ├─ 차트 7종 PNG (charts\ 폴더)
  └─ xlsx 3종 생성
```

## 6. 핵심 기술 이슈 및 해결

| 이슈 | 해결 방법 |
|------|----------|
| `사업계획_수주_List.csv` 상단 9행이 헤더(환율·구성비) | `skiprows=list(range(9))` 적용 → 계획 합계 2,607.5억 일치 확인 |
| `실적_매출예상.csv` 상단 4행이 환율 헤더 | `skiprows=list(range(4))` 적용 |
| EUC-KR / UTF-8 인코딩 혼재 | chardet로 파일별 자동 감지 후 로드 |
| 팀명 형식 불일치 (`01.국내1` 등) | 정규식 `re.sub(r'^\d+\.', '')` 으로 앞번호 제거 |
| 프로젝트명 표기 차이 (계획↔실적 간) | rapidfuzz `token_sort_ratio` 퍼지 매칭, 80% 기준 |
| 원본 xlsx DRM 보호로 직접 수정 불가 | CSV 경로 우회 + 수동 입력 안내값 출력 |
